COLA and Overlap-add STFT Processing

Bruce Sharpe

1 Introduction

It is common in audio (and other kinds of signal processing) to
  1. convert the time-domain signal to a spectrogram via the short-time Fourier transform (STFT),
  2. do some processing on the spectrogram,
  3. convert the spectrogram back to the time domain via the inverse short-time Fourier transform (ISTFT).
When creating the spectrogram in Step 1, a window is usually applied to the samples in each frame, and the frames overlap. Because of this, it is not completely obvious how to recover a time domain signal from the spectrogram.
In fact, depending on the processing that is done, there may not be a time-domain signal whose STFT is exactly the output from Step 2. In this note, we are going to ignore this issue and consider only the simple case where we don’t do anything in Step 2. That is, we want to explore under what conditions it is possible to invert the STFT.
The good news in this case is that, given two conditions that are easily achieved in practice, it is possible to exactly invert the STFT.
The first condition is that there is no gap between the frames. This makes intuitive sense because otherwise we are leaving out some samples when doing the STFT, so there is no way to reconstruct them.
The second condition is that for all This is not the weakest form of the condition, but it is sufficient and easily satisfied in practice. However, in the literature, you will see mention of a much stronger condition: the constant-overlap add (COLA) constraint. You can easily get the impression that COLA is necessary for the STFT to be invertible. In this note we derive the invertibility of the ISTFT from first principles and show that COLA, while sufficient, is not necessary.
Recent versions of SciPy include functions for the STFT and ISTFT. This note was prompted by the implementation of the istft function in scipy.signal (v1.0.0) which will check if COLA is satisfied and will abort if not. We want to make the case for removing that check.

6 Discussion and conclusions

We have shown that COLA is sufficient but not necessary for the STFT to be invertible. We were motivated do this by the implementation of the SciPy library function istft. The COLA constraint imposed by that function should be removed or at least made optional.
It should be noted that despite what the documentation implies, there is no COLA checking in the forward function, stft.
